    Whitelist

    You can use a whitelist with Jotbox to store the ID (jti ) of every token that's generated in a database.
    When a token is verified by Jotbox it also verifies that it exists in the whitelist.
    The token is removed from whitelist when it expires (if it has an expiry time)

    This gives you the ability to revoke tokens at any time before they expire.
    The downside is that a database call is triggered every time a token is used but for many applications, that don't run at gigantic scale, this is a perfectly acceptable tradeoff.

    You can also use an idle timeout to automatically revoke tokens that are not accessed for a time period.

    A redis implementation of the whitelist is included, but you can also roll your own

    Use the redis whitelist

    from jotbox.whitelist.redis import RedisWhitelist

    Instantiate from redis URL:

    ...
my_whitelist = RedisWhitelist('redis://localhost')

    If you want more control over how the redis connection pool is created, you can pass in an aredis.StrictRedis instance directly:

    ...
from aredis import StrictRedis

my_redis = StrictRedis(...)
my_whitelist = RedisWhitelist(my_redis)

    You can also set a custom key prefix if you'd like (the default prefix is JB:WHITE:):

    my_whitelist = RedisWhitelist(my_redis, key_prefix="WHITELISTED_TOKEN:")

    Now pass your whitelist when creating the Jotbox instance:

    jot = Jotbox[MyPayload](
    ...,
    whitelist=my_whitelist
)

    Idle timeout

    When using a whitelist, you can set an idle timeout for your tokens.
    This means that if the token is not accessed for a given number of seconds it will expire and can not be used anymore.
    This is useful for example for session tokens, where you may want a login session to have a long or no expiry but still want the user to be automatically logged out after say 1 hour of inactivity.

    To use this you need to pass both idle_timeout argument to Jotbox along with whitelist:

    jot = Jotbox[MyPayload](
    ...,
    idle_timeout=3600, # Revoke after 1 hour of inactivity
    whitelist=my_whitelist
)

    Jotbox will then call the whitelist.touch(...) method every time the token is accessed extending its expiry time until either idle timeout or token expiry time (whichever one comes first)

    Use other storage backends

    You don't have to use redis for your whitelist. You can create a subclass of the abstract class jotbox.whitelist.base.BaseWhitelist and use any database/storage method you want.

    You'll need to implement the add, exists, touch and delete methods.

    Check source code of the redis whitelist for a reference implementation.

    Note that with redis, expiry times are handled by redis itself. With other databases you may need to explicitly check and clean up expired tokens.
